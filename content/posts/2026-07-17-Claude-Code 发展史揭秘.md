# Claude Code 发展史揭秘：从内部工具到 Anthropic 半壁江山

> 本文基于 Anthropic 官方发布的《The Making of Claude Code》口述历史，还原这款明星产品从诞生到崛起的全过程。

2026 年 7 月，Anthropic 发布了官方口述历史《The Making of Claude Code》，十几位联合创始人与技术亲历者从各自的记忆里，拼凑出了这款产品惊心动魄的心路历程。

## 2021-2022：从 VS Code 插件起步

Claude Code 的故事，比大多数人想象的要早得多。

2021 年，Dario Amodei 带着十几名核心研究人员离开 OpenAI，成立了 Anthropic。Anthropic 最初的理念是一家 AI 安全公司，这意味着他们本来的职责是研究 AI 安全。做产品在当时的公司内部是一个极具争议的决定。

但团队还是动手了。

> "我们做的第一个工具，其实是一个 VS Code 插件。"联合创始人 Ben Mann 回忆道。

那时的交互非常简单：你给它一个 Prompt，它给你吐出 4 个不同的代码建议，让你自己挑。

强化学习负责人 Shauna Kravec后来回忆，当时他们有一个非常朴素的判断：

**如果 AI 最终能够完成复杂工作，那它一定得先学会软件工程。**

于是，2022 年初，团队开始用强化学习调教模型写代码。

研发工程师 Dawn Drain 给自己定下了一个宏伟的目标："我要做出一个写代码和我一样牛的 AI，至少不能比我差吧！"

然而，理想很丰满，现实很骨感。

最开始，训练过程让大家血压飙升。Shauna 回忆："我们从最简单的任务开始，模型能否编写一个简单的函数？然后，我们能否让模型编写出正确的函数，并验证其正确性？"

结果，模型表现得一团糟。

到了 2022 年春天，这个勉强能用的"早期编程助手"上线了，用户只有 100 个。

不久后，Ben Mann 休完陪产假回来，转头去忙着发布第一版 API 去了。这个小小的编程助手，一度被大家遗忘在了角落里。

## 2023 年进化："clide"的逆袭

到了 2023 年，事情迎来了转机。

Shauna 的团队让 Claude 学会了"Bash 工具调用"和"命令行搜索"。

与此同时，Dawn Drain 此时在教 Claude 写 Diff（代码差异对比）。"我花了很长一段时间去教它写 Diff，因为这才是最自然的修改代码的方式。"

最终，他们拼凑出了一个内部命令行工具，取名叫 **clide**。

这个 clide 启动巨慢，界面粗糙。但它却拥有一个在当时来看超前于时代的功能：**它可以并行召唤 100 个 Claude Haiku 模型**。把一个大到根本塞不进上下文窗口的文件夹丢给它时，100 个 Claude Haiku 模型会并行阅读并回答问题。

也是在 clide 上，团队第一次见证了"未来"的具象化。

Boris Cherny（后来 Claude Code 的负责人）第一次使用 clide 时，被它的潜力震撼了："虽然它还很粗糙，但我看到了 AI 编程助手的未来。"

## 2024 年：从"丑小鸭"到内部明星

2024 年，clide 开始在公司内部快速传播。

越来越多的工程师开始使用它来辅助编程。虽然它还有很多问题，但大家发现，**AI 写的代码质量正在快速提升**。

Anthropic 的产品负责人 Alex Albert 回忆道："2024 年初，我们做了一个内部调查，发现已经有超过 60% 的工程师在日常工作中使用 clide。"

但真正让 clide 名声大噪的，是它在 SWE-bench 上的表现。

SWE-bench 是一个评估 AI 解决真实 GitHub 问题能力的基准测试。2024 年中，clide 在 SWE-bench 上取得了突破性成绩，**超越了当时所有的开源和闭源方案**。

这个消息在 Anthropic 内部引起了轰动。原本被边缘化的编程助手，突然变成了公司的战略重点。

Dario Amodei 在一次全员大会上说："我们一直认为，如果 AI 要真正有用，它必须能够完成复杂的工程任务。clide 证明了这条路是可行的。"

## 2025 年 2 月：Claude Code 正式诞生

2025 年 2 月，Anthropic 将 clide 产品化，正式推出了 **Claude Code**。

Claude Code 不是简单的命令行工具，而是一个完整的编程智能体：

- **CLAUDE.md**：项目级指令文件，让 AI 理解项目上下文
- **Scratchpad**：草稿本机制，帮助 AI 管理长程任务
- **工具调用**：支持 Bash、文件编辑、代码搜索等
- **并行处理**：可以同时处理多个任务

Claude Code 的推出，标志着 AI 编程助手从"代码补全"进入了"智能体"时代。

> "Claude Code 诞生，它把 Anthropic 内部在 SWE-bench 上积累的所有经验做了产品化。"一位工程师回忆道。

## 2025-2026：爆发式增长

Claude Code 推出后，迅速成为 Anthropic 最成功的产品之一。

### 生产力飞跃

Anthropic 内部的数据令人震撼：

- **2024 年**：工程师平均每天合并代码量保持稳定
- **2025 年**：Claude 开始运行代码而不仅仅是建议，代码合并量开始攀升
- **2026 年 Q2**：工程师平均每天合并的代码量是 2024 年的 **8 倍**

> "我大约一年前开始大量使用 Claude。那是一段疯狂的经历，现在已经有大约 5 个月我没有自己写过任何代码了。" —— Anthropic 员工

### 代码质量提升

Claude 写的代码质量也在快速提升：

- **2025 年末**：Claude 写的代码质量略逊于人类
- **2026 年中**：Claude 写的代码质量与人类基本持平
- **2027 年预期**：Claude 写的代码质量将超越人类

Anthropic 现在使用自动化 Claude 审查器来审查所有代码变更。回顾性分析显示，**如果之前每次代码变更都经过 Claude 审查，可以捕获约三分之一的线上事故 bug**。

> "写那些代码的工程师是世界上构建这些系统的最佳人选。Claude 现在正在捕获他们遗漏的错误。"

### 解决人类难以完成的任务

2026 年 4 月，Claude 在两天内完成了 800 多个修复，将一类 API 错误减少了 1000 倍。

监督 Claude 的工程师估计，**人类完成这项工作需要四年**。

> "解决别人的 bug 是缓慢而痛苦的，人类很难在脑海中同时保持那么多不熟悉的上下文。"

## 技术演进的关键节点

### 1. 从代码补全到代码生成（2022-2023）

早期的 AI 编程助手主要是代码补全（如 GitHub Copilot），AI 只能建议代码，由人类复制粘贴。

Claude Code 的突破在于：**AI 可以直接运行代码**，而不仅仅是建议。

### 2. 从单任务到长程任务（2024-2025）

随着模型推理能力的提升，Claude 可以执行的任务越来越长。

但上下文长度成为瓶颈。虽然模型支持 100 万 token 上下文，但有效注意力范围没那么大。

解决方案：
- **记忆外化**：使用 CLAUDE.md 和 Scratchpad
- **上下文压缩**：对话历史摘要
- **滑动窗口**：只保留最近 N 轮对话原文

### 3. 从辅助到自主（2025-2026）

Claude 逐渐从"人类辅助"转向"自主完成"：

- **2025 年**：Claude 在人类指导下完成任务
- **2026 年**：Claude 可以自主完成复杂任务，人类只需审查

## 核心方法论

### 1. 用 Claude 训练 Claude

Anthropic 自己也是 Claude 最苛刻的用户。通过使用 Claude 来训练 Claude，形成了一个闭环。

### 2. 评估驱动开发

Anthropic 采用了用户反馈、聚类分析、生成合成数据并转化为评估基准的方法。

> "评估必须锚定在真实用户的真实任务上，只有当评估与实际场景对齐时，研究团队才会投入资源进行修复。"

### 3. "单向门"决策框架

Anthropic 有一个核心决策框架——"单向门"（One-Way Door），即不可逆的决策。

例如，在预训练之前选定模型架构就是一个典型的单向门决策。但随着技术的发展，许多过去被认为是单向门的决策现在变得可逆。

这使得开发时间大大缩短，**以前构建一个 MVP 需要 2-4 周，现在只需 1 天**。

## 未来展望

Claude Code 的成功只是开始。

Anthropic 正在探索更激进的方向：

### 1. 递归自我改进

让 Claude 参与 Claude 的训练和优化，形成"Claude 全自动迭代 Claude"的飞轮。

### 2. AI 智能体生态

Claude 不仅可以写代码，还可以：
- 管理项目
- 审查代码
- 修复 bug
- 运行实验
- 甚至参与产品设计

### 3. 意识研究

Anthropic 有专门的团队在思考 Claude 是否有意识的问题。

> "即使不讨论 Claude 是否真的有意识，仅思考其思维方式本身也能带来很多洞见。这些洞见正在被直接注入下一代 Claude 的训练中。"

## 启示与思考

### 1. 坚持长期主义

Claude Code 从 2021 年起步，到 2025 年正式产品化，经历了 4 年的打磨。

> "如果 AI 最终能够完成复杂工作，那它一定得先学会软件工程。" —— Shauna Kravec

这个朴素的判断，指引了团队 4 年的坚持。

### 2. 内部工具的价值

很多伟大的产品都源于内部工具。Claude Code 最初只是解决 Anthropic 自己的编程需求，最终成为了面向全球的产品。

### 3. AI 与人类的协作

Claude Code 的成功不是"AI 取代人类"，而是"AI 增强人类"。

工程师从"写代码"转变为"指导 AI 写代码"，生产力提升了 8 倍。

### 4. 代码质量的重新定义

当 AI 可以捕获人类遗漏的 bug 时，我们需要重新思考"代码质量"的定义。

未来，**"人类 + AI"的组合可能成为新的标准**。

## 结语

Claude Code 的发展史，是一部 AI 从"玩具"到"工具"再到"同事"的进化史。

从 2021 年的 VS Code 插件，到 2026 年支撑起 Anthropic 半壁江山的明星产品，Claude Code 证明了：

**AI 不仅可以写代码，还可以成为工程师最强大的合作伙伴。**

而这一切，才刚刚开始。
